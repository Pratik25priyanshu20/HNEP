# ESOL sanity check — HNEP v0.3.0

**Verdict:** `Inconclusive`

| Probe | Score | CI | Verdict |
|---|---|---|---|
| SurrogationProbe | 0.0749 | [0.0397, 0.1296] | REPLACEABLE (CI straddles 0.099 threshold) |
| InterventionProbe | 0.1266 | [0.0823, 0.1746] | LOAD-BEARING (cleanly above 0.055 threshold) |

Thresholds (v0.3.0 CI-upper-bound calibration):
`Thresholds(SS<0.099 ⇒ REPLACEABLE, Δ≥0.055 ⇒ LOAD-BEARING)`

## Why "Inconclusive" is the right verdict here

SS=0.075 sits below the SS threshold (0.099) at the point estimate, but
the bootstrap CI extends to 0.130 — straddling the threshold. The QCT
classifier emits Inconclusive whenever a CI straddles either threshold,
which is honest about finite-sample uncertainty rather than forcing a
verdict the data don't support.

The Δ side is clean (Δ=0.127, CI=[0.082, 0.175], well above 0.055): the
trained decoder *does* meaningfully use the quantum branch. So the
question HNEP is genuinely uncertain about is whether the quantum branch's
information could have been produced equally well by a purely classical
surrogate — which is precisely the SS/REPLACEABLE question.

This is consistent with the qualitative finding from the Phase B work
preceding the HNEP library: ESOL surrogation was "easy" — the quantum
output was largely reproducible from GNN embeddings via classical
regression. v0.3.0 returns the methodologically correct call:
the model would *probably* be classified REPLACEABLE → REGULARIZER on
more data, but at the current sample size HNEP honestly reports the
verdict is borderline.

## No v0.2 baseline to compare against

The brief asked for a v0.2 → v0.3.0 verdict diff on ESOL. That comparison
isn't structurally possible: **no library-generated v0.2 baseline exists
for ESOL** because the original Phase B analysis used custom scripts
predating the HNEP library. The "HNEP Phase B" extractions in the parent
repo (`experiments/phase_b/extractions/esol_extraction.npz`) are the
inputs to that pre-library analysis; the analysis itself never ran
through `hnep.evaluate`.

Re-running v0.2's library code on the same extraction to manufacture a
baseline isn't worth doing for two reasons:

1. The Inconclusive verdict is the methodologically correct call at this
   sample size. A v0.2 verdict would either agree (Inconclusive — same
   conclusion), or disagree (forced REGULARIZER under v0.2's narrower
   proxy CI that didn't straddle 0.20 — and v0.2's narrower CI was
   itself part of the calibration bias v0.3.0 fixes).
2. Either outcome would be uninformative about whether v0.3.0 is a
   regression; both versions agree the quantum branch is "borderline
   replaceable, clearly load-bearing", which is the underlying chemistry
   finding and the only signal that matters.

## Caveat — Ridge surrogate decoder

The original ESOL hybrid model is a Flax network whose weights live in
the parent thesis repo's checkpoints. Loading them inside HNEP would
require pulling in the parent repo's model definition. This sanity check
substitutes the trained decoder with a Ridge readout fit on
`(quantum, gnn) → predictions` from the precomputed extraction. The
Ridge tracks the original network closely on its training distribution
(R² ≈ 0.98 train, 0.93 test on the recorded predictions), but borderline
verdicts may not be identical to a full Flax re-evaluation. The SS verdict
is exact (it depends only on `(quantum, gnn)`, which we have exactly);
the Δ side is the part where the Ridge approximation matters.
